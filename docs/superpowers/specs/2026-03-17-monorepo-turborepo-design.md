# Monorepo Restructure with Turborepo

**Date:** 2026-03-17
**Status:** Approved

## Goal

Restructure the BitCompass repository as a proper Turborepo monorepo with one app (`apps/webapp`) and one package (`packages/bitcompass-cli`, no source code changes).

## Repository Structure

```
bitcompass/                        ← workspace root (no app code here)
├── turbo.json                     ← Turborepo pipeline config
├── package.json                   ← workspace root (workspaces: ["apps/*", "packages/*"])
├── bun.lock                       ← single lockfile for all workspaces (regenerated by bun install)
├── README.md                      ← stays at root (repo-level)
├── CLAUDE.md                      ← stays at root (repo-level)
├── .gitignore                     ← updated to add .turbo
├── docs/                          ← stays at root (repo-level: glossary, MCP-CLI-INTEGRATION, superpowers)
├── apps/
│   └── webapp/                    ← moved from root
│       ├── src/
│       ├── public/
│       ├── supabase/
│       ├── index.html
│       ├── vite.config.ts
│       ├── tsconfig.json
│       ├── tsconfig.app.json
│       ├── tsconfig.node.json
│       ├── tailwind.config.ts
│       ├── postcss.config.js
│       ├── eslint.config.js
│       ├── components.json
│       ├── vercel.json
│       ├── vitest.config.ts
│       └── package.json           ← name: "@bitcompass/webapp"
└── packages/
    └── bitcompass-cli/            ← no source code changes
```

## Turborepo Pipeline (`turbo.json`)

```json
{
  "$schema": "https://turbo.build/schema.json",
  "tasks": {
    "build": {
      "dependsOn": ["^build"],
      "outputs": ["dist/**"]
    },
    "dev": {
      "cache": false,
      "persistent": true
    },
    "lint": {},
    "test": {
      "outputs": []
    }
  }
}
```

## Root `package.json`

The root `package.json` is a workspace root only — it has no application code and intentionally omits `"type": "module"`. `turbo` is pinned to `^2` (not `latest`) for reproducibility.

```json
{
  "name": "bitcompass-monorepo",
  "private": true,
  "workspaces": ["apps/*", "packages/*"],
  "scripts": {
    "dev": "turbo dev",
    "build": "turbo build",
    "lint": "turbo lint",
    "test": "turbo test",
    "sonar:fetch": "bunx sonarflow fetch"
  },
  "devDependencies": {
    "turbo": "^2"
  }
}
```

## `apps/webapp/package.json`

Update the existing webapp `package.json` as follows:
- Rename `"name"` to `"@bitcompass/webapp"`
- Remove the `sonar:fetch` script (it moves to the root `package.json`)
- All other dependencies, devDependencies, and scripts remain unchanged (`build:dev`, `preview`, `test:watch`, etc.)

## Migration Steps

1. Create `apps/webapp/` directory.
2. Move webapp files into `apps/webapp/`:
   - `src/`, `public/`, `supabase/`, `index.html`
   - `vite.config.ts`, `tsconfig.json`, `tsconfig.app.json`, `tsconfig.node.json`
   - `tailwind.config.ts`, `postcss.config.js`, `eslint.config.js`
   - `components.json`, `vercel.json`, `vitest.config.ts`
   - `package.json` → becomes `apps/webapp/package.json`
3. In `apps/webapp/package.json`: set `"name"` to `"@bitcompass/webapp"` and remove `sonar:fetch`.
4. Replace root `package.json` with the workspace root version above.
5. Add `turbo.json` at root.
6. Update `.gitignore` to add `.turbo`.
7. Run `bun install` at root to regenerate the root lockfile and link all workspaces.

**Files that stay at root:** `README.md`, `CLAUDE.md`, `docs/`, `.gitignore`, `bun.lock`.

## Notes

**Lockfile format:** The current root uses `bun.lockb` (binary format). After running `bun install` at root, Bun will regenerate a lockfile — the format (`bun.lockb` vs `bun.lock`) depends on the installed Bun version; v1.1+ generates the text-format `bun.lock`. The CLI package has its own `packages/bitcompass-cli/bun.lock` (text format), which will be superseded by the root lockfile after workspace installation. This is expected monorepo behavior and does not constitute a source code change to the CLI.

**`packages/bitcompass-cli/` and workspaces:** Including `packages/*` in the workspace glob means `bun install` at root absorbs the CLI's dependency management into the root lockfile. No `package.json`, scripts, or source files in the CLI are modified. Note: the CLI's `package.json` uses the unscoped name `"bitcompass"` — this is fine for now since no workspace dependency between the webapp and CLI is introduced. If that changes in the future, the CLI should be renamed to a scoped package (e.g. `@bitcompass/cli`).

**`vercel.json` and Vercel deployment:** `vercel.json` moves to `apps/webapp/`. Since Vercel reads this file relative to the project root directory, the Vercel project settings must have `rootDirectory` set to `apps/webapp`. The existing `vercel.json` content (SPA rewrite rule) is unchanged.

**`supabase/` directory:** Moves to `apps/webapp/supabase/`. Any Supabase CLI commands (e.g., `supabase db push`) must be run from `apps/webapp/` after the migration.

**No CI to update:** There is no `.github/` directory in this repo, so no CI pipeline changes are needed.
